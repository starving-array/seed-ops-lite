# Feature Backlog (v3.1.x Evolution)

This backlog prioritizes feature requests for the next development cycles.

---

## 1. Must Have (High Priority)
- **Shared Connection Pools**: Centralized platform repository helpers to reduce connection overhead in high-throughput environments.
- **Notification Webhooks**: Built-in webhook targets in `NotificationManager` for Slack, Teams, and Email notifications.

---

## 2. Should Have (Medium Priority)
- **Distributed Lock Managers**: Coordinate multi-worker state updates using distributed locking mechanisms.
- **Workflow State Restarts**: Finer granularity controls to allow restarting workflow steps from arbitrary steps rather than from the stage checkpoint alone.
